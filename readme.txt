Модуль новостей.

Описание

Содержит бек и фронт. Если использовать указанные в этой инструкции правила разбора URL, то адрес фронта - 
<адрес_приложения>/news. Адрес бэка - <адрес_приложения>/news/admin

Установка

1) Папку модуля нужно разместить в "<папка_приложения>/modules". В общей публичной папке сайта нужно
создать папку images.

2) Для работы модуля необходимо использование базы данных.
Если в приложении, к которому подключается модуль, не используется никакой БД, 
то её нужно создать. В файле "createDbMySQL.sql" содержаться нужные для этого инструкции для СУБД MySQL. 
Этот файл находится в папке модуля.
Для повышения безопасности не забудьте заменить пример пароля пользователя СУБД, который есть в этом 
файле, на свой уникальный пароль, и не добавляйте его под контроль версий. 
Можно импортировать этот файл в MySQL целиком, но тогда кроме создания БД будут созданы и заполнены 
данными примеров таблицы. Если нужно только создать БД (и, возможно, пользователя СУБД для работы с этой 
базой), то следует открыть sql-файл и взять из него только нужные sql-запросы. 
Возможно использование не MySQL, а другой СУБД. В этом случае шаги будут такими же, как в 
этой инструкции. Но на некоторых шагах нужно будет вносить изменения. На данном шаге может потребоваться 
изменение sql-запросов, поскольку реализации языка sql в разных СУБД различаются.

3) Если это ещё не сделано, в приложении нужно настроить подключение к БД.

4) Модуль нужно подключить в конфигурации приложения. Например:
'modules' => [
    'news' => [
        'class' => 'app\modules\news\News',
    ],
]

5) Там же в конфигурации приложения нужно задать правила разбора URL для модуля. Для этого в
'urlManager'['rules'] нужно добавить:
'news'                       => 'news/default/index', 
'news/view/<id:\d+>'         => 'news/default/view',
'news/admin'                 => 'news/admin/index',
'news/admin/view/<id:\d+>'   => 'news/admin/view',
'news/admin/update/<id:\d+>' => 'news/admin/update',
'news/admin/delete/<id:\d+>' => 'news/admin/delete',

6) Создание таблиц в БД.
Если нужные для работы модуля таблицы не были созданы sql-запросами на шаге 2, то их можно создать с 
помощью миграций. Это более универсальный способ, который подходит не только для MySQL, но и для
других СУБД.
Миграции модуля находятся в папке "<папка_модуля>/migrations". 
Чтобы миграции модуля учитывались отдельно (и информация о них сохранялась в отдельной таблице БД), 
нужно сконфигурировать отдельную команду миграции для модуля. Для этого в конфигурации консольной части 
приложения в массив 'controllerMap' нужно добавить конфигурацию этой команды. Например:
// Миграции модуля News
'migrate-news' => [
    'class' => 'yii\console\controllers\MigrateController',
    'migrationNamespaces' => ['app\modules\news\migrations'],
    'migrationTable' => 'migration_news',
]
Тогда для работы с миграциями модуля можно будет применять консольную команду yii migrate-news.

7) Если данные примеров нужны, и они не были добавлены на шаге 2, то нужно добавить их сейчас.

8) Если используются примеры данных, в созданную на шаге 1 папку images нужно положить картинки, 
названия которых совпадают с названиями в БД.
